# architecture-decision-records

ADRs — one file per significant decision, Context/Decision/Consequences template
(TDD §4.5).

Every stack choice "locked in" by the Implementation Roadmap that the source docs left
open (e.g. Turborepo over Nx, BullMQ over SQS for MVP, ECS over K8s for MVP) should
have a corresponding ADR here — the roadmap document explains the reasoning narratively,
but ADRs are the durable, individually-referenceable record engineers actually link to
in PRs.

No ADRs exist yet — none of the M0 stack decisions have been formally recorded here,
which is a gap worth closing before M1 starts (flagged previously in the M0
verification notes).
