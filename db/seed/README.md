# seed

Seed data required for a freshly provisioned tenant to be usable: default roles and
the permission catalog (iam.roles / iam.permissions), pipeline stage templates, and
any other fixture data the schema doc or UX spec assumes exists by default rather than
being created per-tenant on first use.

**Status:** the seed mechanism is configured and genuinely runs (`index.ts`, invoked
via `pnpm --filter @elewate/core-api run db:seed`, wired through
`services/core-api/prisma.config.ts`'s `migrations.seed`), but it's a real no-op right
now — `db/schema.prisma` has zero models, so there's no data to seed yet. Verified by
actually running it against a live PostgreSQL instance, connecting for real, and
logging that there's nothing to do — not just asserted.

Written against the raw `pg` driver rather than the generated Prisma Client, since the
client can only be generated by `prisma generate`, which needs a binary download this
was authored without network access to (see `services/core-api/README.md` for the
exact error). Upgrade this file to use the generated `PrismaClient` once that's been
run for real and actual models exist to seed.
